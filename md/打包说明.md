1. 安装依赖

```powershell
uv sync
```

2. 执行打包

```powershell
uv run pyinstaller --noconfirm --clean -F -w --name image_compressor --upx-dir upx --optimize 2 --collect-all py7zr image_compressor.py
```

推荐原因

参数 | 作用
--- | ---
`uv run pyinstaller` | 直接使用项目虚拟环境里的 PyInstaller，避免命令跑到全局 Python
`--noconfirm` | 自动覆盖旧的 `build/`、`dist/` 等输出目录
`--clean` | 清掉上一次打包缓存，减少脏缓存导致的奇怪问题
`-F` | 打包成单文件 `exe`
`-w` | GUI 模式，不弹黑框
`--name image_compressor` | 固定生成文件名，避免跟脚本名或历史产物混淆
`--upx-dir upx` | 使用 `upx` 目录里的 UPX 压缩；如果本机没有 UPX，可以直接删掉这个参数
`--optimize 2` | 使用较高优化等级
`--collect-all py7zr` | 把 `py7zr` 及其运行所需资源一起收进去，避免运行时解压 7z 失败
`image_compressor.py` | 主入口脚本

如果你不用 `uv`，也可以执行：

```powershell
python -m PyInstaller --noconfirm --clean -F -w --name image_compressor --upx-dir upx --optimize 2 --collect-all py7zr image_compressor.py
```

打包完成后，产物默认在 `dist\image_compressor.exe`。
